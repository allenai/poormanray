# Release Notes (0.7.0)

## New Features
- Add Google Cloud Platform (GCP) support: `--cloud gcp` flag enables managing GCE instances and GCS buckets alongside existing AWS/EC2/S3 support.
- New `--cloud` option (default: `aws`, env: `PMR_CLOUD`) on all commands to select the cloud provider.
- New `--gcp-project` option (env: `GCP_PROJECT`) for specifying the GCP project ID.
- New `gcp_instance.py` backend module implementing `InstanceInfo`, `InstanceStatus`, `BucketInfo`, and `ClientUtils` for GCP.
- GCP instances use service account credentials by default — no credential push needed during `setup`.
- `GCS_TOOLS_SETUP` script added to `commands.py` for installing `gcloud` CLI on remote instances.

## Changes
- `--ami-id` renamed to `--image` (accepts AMI IDs for AWS and image families for GCP).
- `--instance-username` default is now cloud-aware: `ec2-user` for AWS, `owner` for GCP.
- `--region` default is now cloud-aware: `us-east-1` for AWS, `us-central1` for GCP.
- `--storage-type` is now a free-form string (was a fixed `Choice`) to support both AWS and GCP disk types.
- GCP added as an optional dependency group: `pip install poormanray[gcp]`.
- `ssh_session.py` `Session` class accepts `cloud` and `gcp_project` parameters.
