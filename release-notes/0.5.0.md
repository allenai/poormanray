# Release Notes (0.5.0)

## New Features

- Added `create_bucket`, `update_bucket`, and `delete_bucket` commands to manage S3 buckets directly from `pmr`.
- Added `update_cluster` to backfill missing cluster tags on EC2 instances without overwriting existing values.

## Changes

- `delete_bucket` now prompts for confirmation before deleting; use `-y`/`--yes` to skip.
- Bucket and cluster commands no longer expose irrelevant instance options (e.g. `--ssh-key-path`, `--instance-type`) in `--help`.
- S3 bucket name validation is enforced on `create_bucket` (AWS naming rules: 3-63 chars, lowercase, no consecutive periods).

## Housekeeping

- Split `common_cli_options` into `base_cli_options` (name, project, region, owner) and `common_cli_options` (adds instance-specific flags). Bucket and cluster commands now use the lighter `base_cli_options`.
