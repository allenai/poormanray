# Release Notes (1.0.0)

## New Features
- Add Google Cloud Platform (GCP) support: `--cloud gcp` flag enables managing GCE instances and GCS buckets alongside existing AWS/EC2/S3 support.
- New `--cloud` option (default: `aws`, env: `PMR_CLOUD`) available on all commands to select the cloud provider.
- New `--gcp-project` option (env: `GCP_PROJECT`) for specifying the GCP project ID.
- New `gcp_instance.py` backend module implementing `InstanceInfo`, `InstanceStatus`, `BucketInfo`, and `ClientUtils` for GCP.
- GCP instances use service account credentials by default — no credential push needed during `setup`.
- `GCS_TOOLS_SETUP` script added to `commands.py` for installing `gcloud` CLI on remote instances.
- Added `create_bucket`, `update_bucket`, and `delete_bucket` commands to manage S3/GCS buckets directly from `pmr`.
- Added `update_cluster` to backfill missing cluster tags/labels on instances without overwriting existing values.
- Added `wait` command to poll instances until all status checks pass, with optional readiness command.
- Added `ssh` command to open an interactive SSH session to a running instance with multi-instance selection.
- Added docstrings to all CLI commands.

## Changes
- `--ami-id` renamed to `--image` (`-a` short flag preserved). Accepts AMI IDs for AWS and image families for GCP.
- `--instance-username` default is now cloud-aware: `ec2-user` for AWS, the `--owner` value for GCP.
- `--region` default is now cloud-aware: `us-east-1` for AWS, `us-central1` for GCP.
- `--storage-type` is now a free-form string (was a fixed `Choice`) to support both AWS and GCP disk types.
- GCP dependencies (`google-cloud-compute`, `google-cloud-storage`, `google-auth`) added as optional extras: `pip install poormanray[gcp]`.
- `ssh_session.py` `Session` class accepts `cloud` and `gcp_project` parameters.
- `delete_bucket` prompts for confirmation before deleting; use `-y`/`--yes` to skip.
- Bucket and cluster commands use the lighter `base_cli_options` (no instance-specific flags like `--ssh-key-path`).
- Better formatting for tags using YAML and bold.
- Improved `list` output alignment; GCP instances show a combined `Id/Name` field when both are identical.
- Renamed commands for consistency:
  - `create_bucket` → `create-bucket`
  - `update_bucket` → `update-bucket`
  - `delete_bucket` → `delete-bucket`
  - `update_cluster` → `update-cluster`


## Housekeeping
- Split `common_cli_options` into `base_cli_options` (name, project, region, owner, cloud) and `common_cli_options` (adds instance-specific flags).
- Refactored CLI to use `resolve_backend()` pattern — each command resolves its backend module at runtime instead of importing AWS classes at module level.
- Added `resolve_region()`, `resolve_instance_username()`, and `make_tags()` helpers for cloud-aware defaults.
- Extracted shared code from `aws_instance.py` and `gcp_instance.py` into `base_instance.py`: `InstanceStatus` enum, `InstanceInfoBase` dataclass (display properties), and `BucketInfoBase` (validation and constants).
- Major version bump.
