# Release Notes (1.1.0)

## Changes
- `run --script` and `map --script` now transfer scripts to instances via SFTP instead of base64-encoding them into shell commands. This removes command-length limits and handles arbitrarily large scripts.
- Both `run` and `map` generate a wrapper script (`run.sh`) on the remote machine that executes uploaded files in order. Script filenames are content-hashed to avoid collisions.
- Spindown command generation is unified via `_spindown_command()` with `terminate=True` (for `run`) and `terminate=False` (for `map`).
- New `Session.upload_files()` and `Session.upload_scripts()` methods on the SSH session class provide reusable SFTP upload primitives.

**Full Changelog**: https://github.com/allenai/poormanray/compare/v1.0.0...v1.1.0
