# Release Notes (1.3.0)

## Fixes
- `Session.connect()` now uses exponential retry backoff for SSH connections, starting from a 1 second delay and adding jitter derived from the current delay to reduce banner-related connection failures under load.
- `Session.run*()` and file upload paths now retry with a fresh SSH connection on transient failures, raising `FinalConnectionError` only after all retry attempts are exhausted.
- Instance metadata now uses lowercase AWS tag keys for new resources while still recognizing legacy uppercase keys during cluster lookups and warning when legacy-tagged instances are encountered. GCP instance creation now stores `ai2-project` as a resource-manager tag instead of a label, resolves the correct organization or project tag parent instead of assuming the active project namespace, and keeps `project`, `contact`, and `tool` as labels.
- GCP bucket creation and `update-bucket` now apply `ai2-project` as a resource-manager tag binding after bucket creation instead of leaving it behind as a label, while still preserving the standard bucket labels and lifecycle defaults.
- GCP bucket tag-binding failures now surface the actual exception type and log a traceback instead of collapsing into a bare `KeyError('name')`, and the bucket tag-binding path now validates unexpected Cloud Resource Manager responses explicitly.

## Housekeeping
- Removed unused imports from `utils.py`.
