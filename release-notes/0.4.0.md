# Release Notes (0.4.0)

## New Features
- addes support for Ai2 project using syntax `-n cluster-name@ai2-project` when creating an instance
- `pmr version` command — Print the installed poormanray version.
- `pmr run` now executes commands across instances with a thread pool and prints collected per-instance output at the end; use `--parallelism/-j` to cap concurrent runs (defaults to all selected instances).
- `pmr create`, `pmr terminate`, `pmr pause`, and `pmr resume` now execute per-instance lifecycle operations via a shared thread-pool utility; use `--parallelism/-j` to bound concurrency.


## Fixes
- Fixed path typo in publish workflow (`poorman` -> `poormanray` in version bump step).

## Housekeeping
- Updated README: documented `wait`, `ssh`, and `version` commands; added missing `io2e` storage type; expanded common options table with 8 previously undocumented flags; improved examples.
- Cleaned up `cli.py`: rewrote module docstring to list all 14 commands; fixed 7 function docstrings with incorrect/missing parameters; removed ~50 redundant inline comments.
- Updated commands documentation. 
- Refactored library for easier maintenance.
